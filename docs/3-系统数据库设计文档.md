# AheadShop-Plus 系统数据库设计文档

## 1. 设计规范

### 1.1 命名规范
- 数据库名：`aheadshop_{模块名}`，如 `aheadshop_user`
- 表名：小写 + 下划线，如 `t_user_address`
- 字段名：小写 + 下划线，如 `create_time`
- 主键：`id`，BIGINT，自增
- 逻辑删除：`deleted`，0=未删除，1=已删除
- 公共字段：`create_time`、`update_time`、`create_by`、`update_by`

### 1.2 公共字段（MyBatis-Plus 自动填充）

```java
@TableField(fill = FieldFill.INSERT)
private LocalDateTime createTime;

@TableField(fill = FieldFill.INSERT_UPDATE)
private LocalDateTime updateTime;

@TableLogic
private Integer deleted;
```

## 2. 用户库（aheadshop_user）

### 2.1 用户表 t_user

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | BIGINT | Y | 自增 | 主键 |
| username | VARCHAR(50) | Y | — | 用户名 |
| password | VARCHAR(100) | Y | — | BCrypt 加密密码 |
| nickname | VARCHAR(50) | N | — | 昵称 |
| phone | VARCHAR(20) | N | — | 手机号 |
| email | VARCHAR(100) | N | — | 邮箱 |
| avatar | VARCHAR(255) | N | — | 头像 URL |
| gender | TINYINT | N | 0 | 0=未知 1=男 2=女 |
| status | TINYINT | N | 1 | 0=禁用 1=正常 |
| create_time | DATETIME | Y | NOW() | 创建时间 |
| update_time | DATETIME | Y | NOW() | 更新时间 |
| deleted | TINYINT | N | 0 | 逻辑删除 |

**索引：**
- `uk_username` UNIQUE (username)
- `uk_phone` UNIQUE (phone)
- `idx_email` (email)

```sql
CREATE TABLE t_user (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50)  NOT NULL,
    password    VARCHAR(100) NOT NULL,
    nickname    VARCHAR(50),
    phone       VARCHAR(20),
    email       VARCHAR(100),
    avatar      VARCHAR(255),
    gender      TINYINT DEFAULT 0,
    status      TINYINT DEFAULT 1,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted     TINYINT DEFAULT 0,
    UNIQUE KEY uk_username (username),
    UNIQUE KEY uk_phone (phone),
    KEY idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 4. 订单库（aheadshop_order）

### 4.1 订单主表 t_order

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| order_no | VARCHAR(50) | 订单编号（雪花算法） |
| user_id | BIGINT | 下单用户 |
| total_amount | DECIMAL(10,2) | 订单总金额 |
| pay_amount | DECIMAL(10,2) | 实付金额 |
| discount_amount | DECIMAL(10,2) | 优惠金额 |
| freight_amount | DECIMAL(10,2) | 运费 |
| status | TINYINT | 订单状态（见状态码） |
| pay_type | TINYINT | 1=支付宝 2=微信 |
| pay_time | DATETIME | 支付时间 |
| ship_time | DATETIME | 发货时间 |
| receive_time | DATETIME | 收货时间 |
| receiver_name | VARCHAR(50) | 收货人 |
| receiver_phone | VARCHAR(20) | 收货电话 |
| receiver_address | VARCHAR(300) | 收货地址 |
| remark | VARCHAR(500) | 订单备注 |

**订单状态码：**

| 状态码 | 说明 |
|--------|------|
| 0 | 待支付 |
| 1 | 已支付 |
| 2 | 已发货 |
| 3 | 已收货 |
| 4 | 已完成 |
| 5 | 已取消 |
| 6 | 退款中 |
| 7 | 已退款 |

```sql
CREATE TABLE t_order (
    id               BIGINT AUTO_INCREMENT PRIMARY KEY,
    order_no         VARCHAR(50) NOT NULL,
    user_id          BIGINT NOT NULL,
    total_amount     DECIMAL(10,2) NOT NULL,
    pay_amount       DECIMAL(10,2) NOT NULL,
    discount_amount  DECIMAL(10,2) DEFAULT 0,
    freight_amount   DECIMAL(10,2) DEFAULT 0,
    status           TINYINT DEFAULT 0,
    pay_type         TINYINT,
    pay_time         DATETIME,
    ship_time        DATETIME,
    receive_time     DATETIME,
    receiver_name    VARCHAR(50) NOT NULL,
    receiver_phone   VARCHAR(20) NOT NULL,
    receiver_address VARCHAR(300) NOT NULL,
    remark           VARCHAR(500),
    create_time      DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted          TINYINT DEFAULT 0,
    UNIQUE KEY uk_order_no (order_no),
    KEY idx_user_id (user_id),
    KEY idx_status (status),
    KEY idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单主表';
```

## 6. 数据库分库规划

```
aheadshop_user      — 用户、角色、菜单、地址
aheadshop_product   — 分类、品牌、SPU、SKU、属性、评价
aheadshop_order     — 订单、订单明细、退款、订单日志
aheadshop_pay       — 支付记录
```

每个微服务独占一个数据库，避免跨库 JOIN，通过 API/消息队列进行数据交互。
