# AheadShop-Plus 开发流程与任务规划

## 总体节奏

- 总工期：**40 个工作日**（约 8 周）
- 每日工作量：6-8 小时
- 顺序：基础设施 → 后端核心服务 → 后端业务服务 → 前端开发 → 联调测试 → 部署上线
- 原则：**每天结束时有可运行的成果**，不写一天"空代码"

---

### Day 11：订单服务 — 创建订单

```
任务清单：
□ Entity 层
    - Order extends BaseEntity
    - OrderItem extends BaseEntity（注意：不用逻辑删除，直接物理删除或归档）

□ DTO 层
    - OrderCreateDTO { addressId, remark, cartItemIds[] }
    - OrderSubmitVO { orderNo, totalAmount, payAmount }

□ Mapper 层
    - OrderMapper
    - OrderItemMapper

□ Service 层 — OrderService.createOrder()
    流程（单个事务）：
    1. 通过 Feign 获取已勾选购物车商品
    2. 通过 Feign 批量锁库存（失败则整体失败）
    3. 计算商品总价
    4. 获取收货地址快照（Feign 调 user 服务）
    5. 生成订单号（雪花算法）
    6. 保存 t_order + 批量保存 t_order_item
    7. 通过 Feign 清除购物车已勾选商品
    8. 记录订单日志
    9. 发送延时消息（30min 后检查是否支付）→ RabbitMQ TTL + 死信队列

□ Controller
    - POST /api/order/create — 创建订单

□ 异常处理
    - 库存不足 → 回滚，返回错误
    - 地址不存在 → 回滚
    - 购物车为空 → 提示
```

**产出：** 下单流程跑通

### Day 12：订单服务 — 查询 + 状态流转

```
任务清单：
□ 订单查询
    - GET /api/order/list — 我的订单（分页，按状态筛选）
    - GET /api/order/detail/{orderNo} — 订单详情（含商品明细 + 地址）

    - PUT /api/order/cancel/{orderNo}
        · 校验：状态必须是"待支付"

        · Feign 调 product 释放库存
        · 记录日志

□ 超时自动取消（RabbitMQ 延时队列方案）
    - 下单时发送消息到延时交换机（TTL=30min）
    - 消息过期 → 进入死信队列
    - 消费死信队列：检查订单是否仍为"待支付" → 自动取消 + 释放库存

□ 确认收货
    - PUT /api/order/confirm/{orderNo}
        · 校验：状态必须是"已发货"
        · 更新状态为"已完成"

□ 内部 Feign 接口
    - GET /inner/order/{orderNo} — 订单信息（支付回调时调用）
    - PUT /inner/order/pay-success/{orderNo} — 支付成功更新状态

□ RabbitMQ 配置
    - 延时交换机 + 延时队列（TTL=30min）+ 死信队列 + 消费者
```

**产出：** 订单全生命周期管理可用
