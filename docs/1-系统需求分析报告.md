# AheadShop-Plus 系统需求分析报告

## 1. 项目概述

### 1.1 项目名称
AheadShop-Plus — 小型微服务商城系统

### 1.2 项目背景
构建一个基于微服务架构的小型商城系统，采用前后端分离开发模式，后端使用 Spring Boot + MyBatis-Plus，前端使用 Vue，实现商品展示、用户管理、订单处理、支付等核心电商功能。

### 1.3 项目目标
- 提供完整的 B2C 电商购物流程
- 微服务拆分，各服务独立部署、独立扩展
- 前后端分离，降低耦合，提升开发效率
- 系统具备高可用、可扩展、可观测的能力

## 2. 用户角色分析

| 角色 | 说明 | 核心诉求 |
|------|------|----------|
| 游客 | 未注册用户 | 浏览商品、注册账号 |
| 普通用户 | 已注册消费者 | 购物、下单、支付、查看订单、管理收货地址 |
| 商家/管理员 | 店铺运营者 | 商品管理、订单管理、促销活动、数据统计 |
| 超级管理员 | 系统管理者 | 用户管理、权限管理、系统配置 |

## 3. 功能性需求

### 3.1 用户服务（aheadshop-user）

| 编号 | 功能 | 描述 |
|------|------|------|
| U-01 | 用户注册 | 支持手机号/邮箱注册，密码加密存储 |

| U-04 | 收货地址管理 | 新增/编辑/删除/设置默认收货地址 |
| U-05 | 用户权限管理 | RBAC 角色权限控制 |

### 3.2 商品服务（aheadshop-product）

| 编号 | 功能 | 描述 |
|------|------|------|

| P-05 | 商品搜索 | 基于 Elasticsearch 的全文搜索 |
| P-06 | 商品评价 | 用户对已购商品进行评分和评论 |

### 3.3 购物车服务（aheadshop-cart）

| 编号 | 功能 | 描述 |
|------|------|------|
| C-01 | 添加商品 | 选择 SKU 加入购物车 |
| C-02 | 修改数量 | 调整购物车中商品数量 |
| C-03 | 删除商品 | 从购物车移除商品 |
| C-04 | 购物车查询 | 查看购物车列表、合计金额 |
| C-05 | 选中/取消选中 | 控制结算商品范围 |

### 3.4 订单服务（aheadshop-order）

| 编号 | 功能 | 描述 |
|------|------|------|
| O-01 | 创建订单 | 从购物车选中商品生成订单，扣减库存 |
| O-02 | 订单查询 | 按状态/时间查询订单列表和详情 |
| O-03 | 取消订单 | 超时未支付自动取消，用户主动取消 |
| O-04 | 订单状态流转 | 待支付 → 已支付 → 已发货 → 已收货 → 已完成 |
| O-05 | 申请退款/退货 | 用户发起退款/退货申请 |

### 3.5 支付服务（aheadshop-pay）

| 编号 | 功能 | 描述 |
|------|------|------|
| PY-01 | 在线支付 | 对接支付宝/微信支付沙箱环境 |
| PY-02 | 支付回调 | 接收支付平台异步通知，更新订单状态 |
| PY-03 | 支付查询 | 查询支付状态 |
| PY-04 | 退款处理 | 调用支付平台退款接口 |

### 3.6 搜索服务（aheadshop-search）

| 编号 | 功能 | 描述 |
|------|------|------|
| S-01 | 商品搜索 | 关键词全文检索、分类筛选、价格区间 |
| S-02 | 搜索排序 | 按销量、价格、评分排序 |
| S-03 | 搜索建议 | 输入联想词推荐 |

### 3.7 后台管理服务（aheadshop-admin）

| 编号 | 功能 | 描述 |
|------|------|------|
| A-01 | Dashboard | 销售统计、订单趋势、用户增长 |
| A-02 | 商品管理 | 商品上下架、库存管理 |
| A-03 | 订单管理 | 订单查询、发货、退款审批 |
| A-04 | 用户管理 | 用户列表、禁用/启用 |
| A-05 | 促销管理 | 优惠券发放、满减活动配置 |

## 4. 非功能性需求

### 4.1 性能需求
- 首页接口响应时间 ≤ 200ms
- 商品搜索响应时间 ≤ 500ms
- 系统支持 1000+ 并发用户
- 图片等静态资源走 CDN

### 4.2 安全需求
- 用户密码 BCrypt 加密存储
- 接口鉴权采用 JWT Token
- 敏感接口限流防刷
- SQL 注入、XSS 防护
- HTTPS 通信加密

### 4.3 可用性需求
- 系统可用性 ≥ 99.5%
- 服务支持水平扩展
- 数据库主从读写分离
- 关键服务熔断降级（Sentinel）

### 4.4 可维护性需求
- 统一日志收集（ELK）
- 链路追踪（SkyWalking / Zipkin）
- 配置中心（Nacos）
- 服务注册与发现（Nacos）

## 6. 接口与集成需求

### 6.1 外部接口
- 支付宝开放平台（沙箱）
- 微信支付开放平台（沙箱）
- 阿里云 OSS（文件存储）
- 短信服务（阿里云 SMS）

### 6.2 内部通信
- 服务间同步调用：OpenFeign
- 服务间异步通信：RabbitMQ 消息队列
- 服务注册与发现：Nacos
- 配置管理：Nacos Config
